## Portfolio principles

This repository is built around a few core rules:

### 1. No filler projects
Every project should be:
- portfolio-worthy
- résumé-worthy
- interview-worthy
- role-aligned
- proof-based

### 2. Small but strong
Projects should be:
- short enough to finish
- real enough to matter
- polished enough to showcase

### 3. Real work first
Whenever possible, this portfolio should include:
- real Azure resources
- real troubleshooting cases
- real monitoring and log work
- real identity and access scenarios
- real documentation and runbooks
- real tools and helper systems

### 4. Tools matter
This portfolio is also meant to show exposure to many tools and software platforms, including:
- Azure Portal
- Azure CLI
- Cloud Shell
- GitHub
- VS Code
- PowerShell / Bash
- draw.io
- monitoring and log tools
- identity and access admin tools
- documentation and diagram tools

### 5. Output over theory
A unit is considered complete only when it produces:
- a usable artifact
- proof or evidence
- a writeup
- clear role relevance
